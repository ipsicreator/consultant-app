# 수프리마 플랫폼 메뉴별 교육적 의미 설계안

## 목적
이 문서는 앱의 각 메뉴가 단순 기능이 아니라, 학생 성장과 진학 성과에 어떤 교육적 가치를 만드는지 명확히 정의합니다.

---

## 0. 입력 전 (준비 화면)
- 메뉴/화면명: `진단 준비`
- 교육적 의미:
  - 학생의 현재 상태를 정확히 파악하기 위한 출발점
  - 목표 전공/진로와 현재 데이터 간의 간극을 진단할 준비 단계
- 핵심 요소:
  - 수집할 데이터 체크리스트(학생부, 성적, 활동 기록)
  - 개인정보/기록 활용 동의 확인
  - 분석 목표 설정(내신 개선, 세특 강화, 탐구 포트폴리오)

## 1. 입력 (학생 데이터 입력)
- 메뉴/화면명: `학생 입력`
- 교육적 의미:
  - 정량(성적) + 정성(활동/서술) 데이터를 함께 입력해 편향 없는 진단 가능
  - 학생 맞춤형 지도 근거를 누적하는 단계
- 핵심 요소:
  - 학생 기본정보, 학교/학년, 목표 계열
  - 학생부 텍스트/자료 입력
  - 탐구 관심사/강점/보완점 입력

## 2. 입력 중 (처리/진행상태)
- 메뉴/화면명: `분석 진행`
- 교육적 의미:
  - 분석이 어떻게 진행되는지 투명하게 보여 상담 신뢰를 높임
  - AI 결과를 무비판 수용하지 않고 검토 가능한 절차 확보
- 핵심 요소:
  - 단계별 진행률(파싱 → 구조화 → 진단 → 제안)
  - 오류/누락 데이터 안내
  - 재시도/수정 입력 동선

## 3. 입력 후 결과 요약
- 메뉴/화면명: `결과 요약`
- 교육적 의미:
  - 학생과 보호자가 빠르게 이해할 수 있는 핵심 진단 제공
  - 현재 수준과 목표 수준의 차이를 명확히 시각화
- 핵심 요소:
  - 강점/보완점 3줄 요약
  - 과목별/활동별 우선순위
  - 즉시 실행 가능한 다음 행동 제안

## 4. 결과 상세/편집
- 메뉴/화면명: `상세 분석`
- 교육적 의미:
  - AI 결과를 교사가 교육적으로 해석/수정해 실제 지도안으로 전환
  - 학생 개별성 반영(학습 습관, 학교 맥락, 과목 특성)
- 핵심 요소:
  - 항목별 근거 확인
  - 문장 편집/코멘트
  - 세특 초안 품질 개선(표현, 근거, 연계성)

## 5. 경과/히스토리
- 메뉴/화면명: `경과 추적`
- 교육적 의미:
  - 단발성 상담이 아니라 성장 과정 중심 관리
  - 학생 변화(성적/활동/서술 역량) 추적을 통한 피드백 강화
- 핵심 요소:
  - 시점별 결과 비교
  - 개선률/정체 구간 표시
  - 지도 개입 기록과 결과 연동

## 6. 최종 산출 (리포트)
- 메뉴/화면명: `리포트 출력`
- 교육적 의미:
  - 상담 내용을 학생·보호자·교사가 함께 활용 가능한 학습 문서로 전환
  - 다음 학기/다음 평가를 위한 실행 계획 문서화
- 핵심 요소:
  - 요약 리포트 + 상세 부록
  - 실행계획(주차별 목표/점검항목)
  - 인쇄/PDF 공유

---

## 기존 메뉴와 매핑 권장안
- `학생 CRM 관리` → `학생 입력` + `경과 추적`의 허브
- `학생부 분석` → `결과 요약`/`상세 분석`
- `AI 채팅창`(기존 탐구 브레인) → `입력 후 결과 요약` 보조 및 탐구 확장
- `학습/입시 플래너` → `최종 산출`과 실행계획 연결
- `관리자 배정 관리` → 운영 메뉴(교육 행정)
- `환경 설정` → 템플릿/라이브러리/평가기준 관리

## 적용 원칙
1. 모든 메뉴는 "학생 성장 근거"를 남겨야 함
2. 모든 결과는 "다음 행동"으로 연결되어야 함
3. 모든 기록은 "시점 비교"가 가능해야 함
4. 출력물은 "학생/보호자 설명 가능성"을 기준으로 작성
